  <action>
Create `src/components/ResultsTable.tsx` as a named-export `'use client'` component. Use the full skeleton from the research document as the implementation baseline. Key requirements repeated here for fidelity:

**Component signature:**
```typescript
interface ResultsTableProps {
  rows: ResultRow[];
  clientName: string;
}

export function ResultsTable({ rows, clientName }: ResultsTableProps) { ... }
```

**Return null when empty:** `if (rows.length === 0) return null;` — placed before any hooks that depend on rows length. Because hooks must not be conditional, place all hooks before this guard and ensure hooks receive `rows` directly. The guard comes after all hook declarations.

**Sort state:** `const [sortDir, setSortDir] = useState<SortDir>('desc');` — default descending (highest score first, per locked decision). `sortedRows` is a `useMemo` of `[...rows].sort(...)`.

**Virtualizer:** `useVirtualizer` receives `count: sortedRows.length` (sorted copy, not original `rows`). `estimateSize: () => 44`. `overscan: 5`.

**Table layout — CRITICAL pitfalls to avoid:**
- `<table>` must have `style={{ tableLayout: 'fixed', width: '100%', borderCollapse: 'collapse' }}`
- `<colgroup>` with 6 `<col>` elements is REQUIRED — without it, absolute-positioned `<tr>` elements collapse all columns to zero width. Use these widths:
  ```
  Original Name:    22%
  Entity Type:      12%
  Linguistic Region: 16%
  Degraded Variant: 22%
  Rule Applied:     18%
  Score:            10%
  ```
- `<thead>` must have `className="sticky top-0 z-10"` AND an explicit background color (`bg-white` or equivalent). Without the background the sticky header is transparent and rows scroll visibly beneath it. Use Tailwind: `className="sticky top-0 z-10 bg-white"`.
- `<tbody>` must have `style={{ height: \`${virtualizer.getTotalSize()}px\`, position: 'relative' }}`. This is what gives the scrollbar its correct proportional size.
- Each `<tr>` in the virtual items map must have `style={{ position: 'absolute', top: 0, left: 0, width: '100%', transform: \`translateY(${virtualRow.start}px)\`, height: \`${virtualRow.size}px\` }}`. Without `height: virtualRow.size + 'px'` rows collapse.

**Score column header:** Clickable. `onClick={() => setSortDir(d => d === 'desc' ? 'asc' : 'desc')}`. Display arrow: `sortDir === 'desc' ? '↓' : '↑'`. Add `cursor-pointer select-none` Tailwind classes on the `<th>`.

**Score cell display:** `{Math.round(row.similarityScore * 100)}% {row.caught ? '✓' : '✗'}`. Use `font-mono` for alignment.

**Scroll container:** `<div ref={parentRef} style={{ height: '600px', overflowY: 'auto' }}>`. The fixed height is a locked decision. Add `className="rounded-md border"` for visual containment.

**Summary row (above table, below form):**
- Outer `<div className="flex items-center justify-between">`
- Left: `<p>` showing catch-rate sentence. Use `computeCatchRate(rows)` (unsorted original rows). Use `DEFAULT_CATCH_THRESHOLD` from `@/lib/constants` to derive threshold percentage: `Math.round(DEFAULT_CATCH_THRESHOLD * 100)`. Sentence: `{caught} of {total} degraded variants ({percent}%) would be caught at the {threshold}% match threshold`
- Right: `<Button variant="outline" size="sm" onClick={handleDownload} disabled={rows.length === 0}>Download CSV</Button>`

**Download handler:** `const handleDownload = () => { triggerCsvDownload(buildCsvString(rows), buildCsvFilename(clientName)); }` — uses unsorted original `rows` (download full dataset, not just the sorted view). CSV string is built inside the onClick, not during render.

**Outer wrapper:** `<div className="mx-auto max-w-5xl space-y-3 px-4">` — `max-w-5xl` is the locked decision for table container width.
