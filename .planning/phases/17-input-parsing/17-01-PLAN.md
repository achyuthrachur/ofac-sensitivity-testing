    **parsePaste:**
    - Input: `'Alice\nBob\nCarol'` (newline-separated) → names: `['Alice', 'Bob', 'Carol']`
    - Input: `'Alice,Bob,Carol'` (comma-separated) → names: `['Alice', 'Bob', 'Carol']`
    - Input: `'  Alice  \n  Bob  '` (padded whitespace) → names: `['Alice', 'Bob']` (trimmed)

    **validateNames:**
    - 10,001 distinct names → `error` field set, `names: []`
    - Input with duplicate `['Alice', 'Bob', 'Alice']` → `names: ['Alice', 'Bob']`, warning with `row: 3` and message containing "duplicate"
    - Input with oversized entry (201 chars) → that entry excluded, warning with `kind: 'oversized'`
    - Empty strings in candidates → silently excluded (warning with `kind: 'empty'`), no crash

    GREEN phase — implement `src/lib/screening/parseInput.ts`:

    Export these types and functions:
    ```typescript
    export interface ValidationWarning {
      row: number;        // 1-based
      message: string;   // "Row N: specific instruction"
      kind: 'empty' | 'duplicate' | 'oversized' | 'multi-column';
    }

    export interface ParseResult {
      names: string[];
      warnings: ValidationWarning[];
      rawCount: number;   // rows before dedup/filter
      error?: string;     // blocking error — if set, names is []
    }

    export function parseCsv(text: string): ParseResult
    export function parseExcel(buffer: ArrayBuffer): ParseResult
    export function parsePaste(raw: string): ParseResult
    export function validateNames(candidates: string[]): Omit<ParseResult, 'rawCount'>
    ```

    Implementation rules:
    - `import * as XLSX from 'xlsx'` at the top (no dynamic import — pure module, not a React component)
    - `import { MAX_SCREENING_NAMES } from '@/types/screening'`
    - parseCsv: split on `/\r?\n/`, split each line on `,`, trim cells, detect header row using conservative keyword heuristic (check if first cell lowercased contains any of: 'name', 'client', 'entity', 'company', 'full name'). If multi-column detected (any row has >1 non-empty cell), use column index 0 and add multi-column warning once. Pass extracted strings to validateNames.
    - parseExcel: `const wb = XLSX.read(buffer)` inside try/catch. On throw, return `{ names: [], warnings: [], rawCount: 0, error: 'File could not be parsed as an Excel workbook. Please upload a valid .xlsx file.' }`. On success: `XLSX.utils.sheet_to_json<string[]>(sheet, { header: 1, defval: '' })` to get rows, then apply same column-detection and header logic as parseCsv. Pass to validateNames.
    - parsePaste: split on `/[\n,]/`, trim, filter empty — pass trimmed strings to validateNames. rawCount = split length before filter.
    - validateNames: blocking check AFTER dedup (if names.length > MAX_SCREENING_NAMES, return error). Per-row: skip empty (warning kind:'empty'), skip oversized >200 chars (warning kind:'oversized'), skip duplicate case-insensitively using Set (warning kind:'duplicate'). Row numbers are 1-based from the input candidates array index.
