## Decisions Made

1. **Scrollspy implementation:** IntersectionObserver with `rootMargin: '-20% 0px -70% 0px'` triggers active state when a section is between 20% from top and 30% from bottom of viewport — reliable without any scroll event listeners.

2. **Quick Start intro placement:** The two introductory paragraphs ("two modes: Sensitivity Test / Screening Mode") are rendered inside `OfacContextSection` above the `<section id="ofac-context">` element, separated from the section heading by a `border-b` divider. This keeps the component self-contained while providing page-level orientation before any anchor is scrolled to.

3. **Secondary CTA positioning:** Wrapped in `<div className="mt-4">` (block element) to force it below the `inline-flex` primary CTA button in the `text-center` parent. A plain `inline-flex` would have floated beside the button on wide viewports.

4. **Header restructure:** Right side changed from a single `<span>` to a `<nav className="flex items-center gap-6">` containing both the User Guide `<Link>` and the existing title span. Preserves the `justify-between` balance (Crowe left, nav group right) without changing header height.

5. **No new dependencies:** All content is pure JSX/prose. IntersectionObserver is a native browser API — no library install. No imports from src/lib/scorer.ts, src/lib/sampler.ts, or src/types/screening.ts in any guide file.
