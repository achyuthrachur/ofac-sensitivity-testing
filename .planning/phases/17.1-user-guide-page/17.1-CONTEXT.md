# Phase 17.1: User Guide Page — Context

**Gathered:** 2026-03-07
**Status:** Ready for planning
**Source:** User discussion

<domain>
## Phase Boundary

A new `/guide` route that sits between the landing page (`/`) and the tool (`/tool`). Static documentation-style page explaining every part of the tool to someone who has never used it before. No new engine features — pure content and layout.

</domain>

<decisions>
## Implementation Decisions

### Route
- **URL:** `/guide`
- **Linked from:** Landing page nav + landing page CTA area
- **Navigation:** Accessible from site header nav alongside existing links

### Layout
- **Sidebar + content** — left nav with section anchor links, content on the right
- Documentation style (like a product manual)
- Good for dense reference material; users can jump directly to any section
- Desktop-first (consistent with rest of app)

### Content Sections (all four required)
1. **OFAC Compliance Context** — what the tool is for, why sensitivity testing matters, Cost of Miss concept, what consultants use it for, regulatory background
2. **Sensitivity Test walkthrough** — how to configure and run the degradation demo, entity types, regions, rules, reading results, CSV export
3. **Screening Mode walkthrough** — how to load names (pre-loaded list, upload, paste), run screening, read tier badges, understand match results
4. **Scoring Engine explainer** — why 3 algorithms (JW + DM + TSR), what each catches, how thresholds work, what the 5 risk tiers mean, name-length penalty, NFKD normalization

### Tone
- Written for a consultant who knows OFAC compliance but NOT this specific tool
- Professional, clear, direct — no jargon without explanation
- Structured: each section has a "What it does" + "How to use it" pattern

### Claude's Discretion
- Exact prose content for each section
- Whether to use numbered steps, callout boxes, or prose paragraphs
- Section heading hierarchy
- Whether to include a "Quick Start" intro at the top
- Visual treatment (icons, dividers, callout styles)
- Whether to show the sidebar as sticky on scroll

</decisions>

<specifics>
## Specific Ideas

- User said: "someone who was not involved in the build at all needs to be able to pick it up and use it immediately"
- The guide should function as a standalone onboarding document — not just reference material
- Should explain WHY decisions were made (e.g., why 3 algorithms, why these thresholds) not just what exists
- The Sensitivity Test section should explain the degradation concepts (what is a character substitution? what does Jaro-Winkler measure?)

</specifics>

<deferred>
## Deferred Ideas

- Interactive demos embedded in the guide (out of scope — static page only)
- Simulation Mode walkthrough (feature not yet built — Phase 21+)
- PDF download of the guide (out of scope for this phase)

</deferred>
